The Code for displaying the Palindrome result is structured into main 3 parts (or functions):

getWord()

Here, we first check to see if the user has entered a value, if not we then prompt them to (lines 6 and 7). Next, we obtain the value of the user input via the ".value" property of the getElementById() function. Next on lines 10, 11 and 12 we search for any characters within the input that are not letters and remove them. We also "normalise" or make all the characters the same case for comparitive purposes.

The string.split() method used on the string provided by the user provides an array (line 15), with each index of the array representing a character in the string. This allows us to create another array in the next step with each index of the current array reversed. We can then later turn them back into strings by using the .join() function on the array.

createBackwardsArray()
Here, lines 22, 23 and 24 we create an index counter for the new array initialized at line 27 to ensure the new array is populated with the same amount of elements in forwardsArray()). Lines 25-28 we populate the new array with the current array's values reversed.

Now that we have 2 arrays with the users input reading forwards and backwards, we hand off to the final part of the process to display the content.

displayPalindrome()
Firstly, on lines 41 and 42, we convert the arrays back to strings in order to prepare them for displaying. Based on whether or not the strings match (the Palindrome requirement) at lines 45 and 52, we use an if/else clause to branch the final operation out into two seperate sets of screen output details.

The methods for displaying the content are the same in both cases (only the words differ); we simply add text to the innerHTML properties of the elements beneath the Results heading, by using the getElementById() function to gain access to the respective element. The invisible class is removed so the affected div element becomes visible and the information displayed within it to the user.
